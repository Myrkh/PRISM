# Docs Assets

Ce dossier recevra les captures finales de la documentation PRISM.

Structure recommandée :
- `front/` : captures des vues utilisateur de l'application
- `engine/` : captures liées au moteur, au contrat, aux résultats et au runtime backend

Règles :
- utiliser des captures PRISM réelles, pas des références externes ou des interfaces d'un autre logiciel
- garder un cadrage cohérent d'une capture à l'autre
- privilégier le `light mode` pour la documentation, sauf cas où le `dark mode` est lui-même l'objet de la démonstration
- éviter les données sensibles, noms client ou informations de site non anonymisées
- quand un chapitre utilise une capture, importer l'asset depuis ce dossier dans le fichier de chapitre concerné
